你有沒有這種經驗？

你請一個朋友幫你寫一封 email 給客戶。他很聰明，打字又快，三分鐘就寫好了。你一看——文字是對的，但語氣不太對，太正式了，不像你平常跟客戶聊天的感覺。

你跟他說：「嗯，可以口語一點嗎？我們跟客戶的風格比較輕鬆～」他馬上改好。

下次你又請他幫忙寫 email。結果？又太正式了。你又要重新解釋一遍。

再下一次？一模一樣 😂

**每次都要從頭教**，是不是很煩？

如果你能寫一份小抄給他，上面寫著：「我們的風格是這樣的，語氣要口語、要加 emoji、結尾要有親切感⋯⋯」然後他每次寫之前先看一下這份小抄——問題不就解決了嗎？

**這份小抄，就是 Skill。**

## Skill 就是一份給 AI 的 SOP 手冊

在前面幾篇文章中，我們聊了 [AI Agent 是什麼](/posts/what-is-ai-agent-vs-chatbot/)、它[怎麼思考](/posts/ai-agent-how-it-thinks/)。你已經知道 Agent 跟普通 ChatBot 最大的差別就是：**它會自己規劃步驟、自己動手做事**。

但你有沒有想過一個問題——Agent 雖然什麼都會做，但它並不知道**你想要怎麼做**。

這就像你請了一個超強的新助理。他能力沒話說——會做簡報、會寫報告、會做研究、會整理檔案。但他第一天上班，不知道你們公司的規矩：

- 報告要用什麼格式？
- email 要用什麼語氣？
- 簡報喜歡什麼風格？
- 做研究要參考哪些資料來源？

所以你每次交代一件事，都要花時間從頭解釋一遍。

**Skill 就是你提前寫好的一份 SOP 手冊**，讓 Agent 在做事之前先讀一遍。讀完之後，它就知道「在你的世界裡，這件事應該怎麼做」。

更白話一點說：

> AI Agent Skill = 結構化的指令，讓 Agent 在特定任務上，用**你想要的方式**來執行。

而且最棒的是——**Skill 就是一份純文字檔案**，裡面用自然語言寫的，不是程式碼。你只要會打字，就能寫 Skill。

## 沒有 Skill vs 有 Skill：差別到底有多大？

講再多都不如直接看對比。

我自己有一個寫作風格的 Skill，裡面定義了我寫部落格文章的風格、語調、結構、格式規範。讓我們看看，**同樣請 AI 幫我寫一段部落格開頭**，有沒有 Skill 的差別：

### 沒有 Skill 的結果

![沒有使用 Skill 的 AI 寫作結果](https://storage.googleapis.com/homuchen.com/images/what-is-ai-agent-skill-1.jpg)

你看，沒有 Skill 的時候，AI 會用一種很「標準」的方式來寫。不能說它寫得不好，但就是⋯⋯**很 AI** 😂 沒有個性、沒有溫度，讀起來像在看維基百科。

### 有 Skill 的結果

![使用寫作風格 Skill 後的 AI 寫作結果](https://storage.googleapis.com/homuchen.com/images/what-is-ai-agent-skill-2.jpg)

差別超大吧！有了寫作風格 Skill 之後，AI 知道我喜歡：

- 用故事開頭
- 語氣要像在跟朋友聊天
- 適度加 emoji
- 結尾要有「掰掰～👋」

**同一個 AI，同一個任務，差別只在於有沒有那份「小抄」。**

這就是為什麼我在[第一篇介紹 Cowork](/posts/claude-cowork-ai-from-chat-to-work/) 的時候就說過：Cowork 本身是個什麼都會的通才，**但加上 Skill，它就變成你領域的專家**。

## Skill 可以用在哪些地方？

Skill 的應用範圍比你想像的廣。基本上，只要你有任何重複性的工作，而且希望 AI 每次都用**一致的方式**來做，就可以寫一個 Skill。

讓我用自己實際在用的 Skill 來舉例：

### 1. 寫作風格

這是最直覺的應用。我自己就有一個寫作風格 Skill，裡面定義了：

- **語調**：口語化、溫暖、像朋友聊天
- **結構**：先講故事再帶概念
- **格式**：怎麼用粗體、引用、emoji
- **禁止事項**：不要用學術語調、不要忘記「掰掰～👋」

每次我請 Cowork 幫我寫文章，它都會先讀這份 Skill，然後用**我的風格**來寫。你現在讀到的這篇文章，背後就有這個 Skill 在運作 😎

### 2. 工作流程

Skill 也能定義完整的工作流程。我有一個關鍵字研究的 Skill，裡面寫好了一套完整的 SEO 研究流程：

1. 先做市場觀察
2. 然後分析競品
3. 做 SERP 分析
4. 建立關鍵字映射
5. 最後做優先級排序

我只要說「幫我做這個主題的關鍵字研究」，Cowork 就會按照這套工作流程一步步做完。不用我每次從頭解釋要怎麼研究、要看哪些東西、要產出什麼格式。

### 3. 專業知識

你還可以把特定領域的專業知識「封裝」進 Skill。例如：

- **SEO 審核 Skill**：定義了檢查文章 SEO 的完整清單——標題有沒有包含關鍵字、描述長度夠不夠、內部連結有沒有做⋯⋯
- **發布文章 Skill**：定義了把文章發布到部落格平台的步驟、需要哪些設定、有哪些注意事項
- **內容策略 Skill**：定義了規劃內容的框架——要考慮什麼、怎麼排優先級、怎麼做系列規劃

這些 Skill 把你的經驗和知識打包起來，讓 AI Agent 可以直接「站在你的肩膀上」做事。

## 你不需要會寫程式

我知道很多人看到「Skill」這個字，第一反應是：「這是不是要寫程式？我不會寫程式怎麼辦？」

完全不需要！😄

AI Agent Skill 的核心就是一份 Markdown 檔案，裡面用**自然語言**寫成。你可以把它想成在寫一份給新同事的工作手冊：

- 這份工作的目標是什麼
- 要注意哪些事情
- 要用什麼風格
- 有哪些步驟
- 什麼該做、什麼不該做

就這樣。不用學任何程式語言，不用裝任何開發工具。

> 如果你會寫文章，你就會寫 Skill。

**不用寫程式就能客製化你的 AI Agent**——這大概是 Skill 最吸引人的地方了。

## 為什麼現在是開始用 Skill 的好時機？

2025 年底，Anthropic 開放了 Skill 的規範，讓 Skill 不只是 Claude 的專屬功能，而是逐漸成為跨平台的開放標準。2026 年初，更推出了 Cowork Plugins，讓 Claude 可以直接連接 Gmail、Google Drive、Microsoft 365 等工具。

這代表什麼？**Skill 正在從「進階玩家才會用的功能」變成「AI 工作的基本配備」。**

現在開始學 Skill，就像十幾年前開始學用手機 app 一樣。你不一定要第一個會，但早點學會，就能早點把那些重複、瑣碎的工作交給 AI 去做。

## 結語

- Skill 可以用在**寫作風格、工作流程、專業知識**各種場景
